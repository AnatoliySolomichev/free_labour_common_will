# Пилот: как позвать знакомых

**Что происходит.** Каждый участник заводит свой блокчейн и записывает в него: что умеет, что ему нужно, чему готов научиться. Общий агрегатор собирает это в один список. Дальше потребности одних закрываются навыками других — сначала вручную, потом всё более автоматически.

Все команды ниже проверены на живом прогоне (5 участников, 2026-07-13).

---

## Часть 1. Организатору — один раз

**1. Агрегатор пилота уже поднят**: `https://free-labour-agg.duckdns.org` (он же общая доска объявлений, релей и раздатчик каталогов; TLS, автозапуск, ежесуточные бэкапы). Поднять свой для другого круга можно так:

```bash
aggregator_server --port 8080 --db ~/aggregator-db --catalog docs/catalogs
```

Без `--catalog` всё работает, но слаги никто не проверяет — а опечатка в слаге делает человека ненаходимым. Указывайте. Наружу такой агрегатор выводите через TLS-прокси (nginx/Caddy), сам он слушает plain HTTP.

**2. Раздать знакомым адрес** — `https://free-labour-agg.duckdns.org`. Всё, что участнику нужно знать.

**3. Стыковать — одной командой:**

```bash
bc match --via https://free-labour-agg.duckdns.org
```

```
СТЫКОВКИ
  + cf971cd3…  "Заменить проводку в двушке"  [срочно]
      закроет e5df6324…  prof.electrician, разряд 4, 1.7 км
  + f13efe7a…  "Нужна помощь с налогами"
      закроет 4bd46c2c…  prof.accountant, разряд 5, удалённо
  - 6fcab8ad…  "Сломалась стиральная машина"  [срочно]
      никто не закрывает

  закрыто 5 из 6

КОЛЬЦА ОБМЕНА (замыкаются без долга наружу)
  cf971cd3… → e5df6324… → f13efe7a… → cf971cd3…

ДЕФИЦИТ И ПЕРЕОБУЧЕНИЕ
  need.appliance-repair  "Сломалась стиральная машина"
      закрыла бы профессия: prof.appliance-repair
      уже хотят это освоить: 6fcab8ad…
```

Отчёт сам проверяет **досягаемость**: сравнивает координаты и радиусы обеих сторон. Удалённая работа (`remote:yes`) географию игнорирует. Недостающие данные никого не прячут: не указан радиус или координата — значит «без ограничений», а не «не подходит».

**Кольца** — самое ценное: замкнутая цепочка помощи, где никто не остался должен наружу.

**4. Если хочется отдать ИИ** — есть сырые данные:

```bash
bc match --via https://free-labour-agg.duckdns.org --json      # готовый отчёт
bc export profiles --via https://free-labour-agg.duckdns.org --out profiles.json   # все профили
```
См. «Промпт для стыковки» ниже.

## Часть 2. Участнику — три шага

### Шаг 1. Завести свою цепь

```bash
bc identity create
```

Создаёт ключи и первую ветку в `~/.blockchain`. **Ключи из папки `keys/` не отдают никому — никогда, ни писарю, ни организатору.** Кто держит ключ, тот и есть вы.

### Шаг 2. Получить черновик и проверить его

Записи за вас может подготовить помощник — писарь, бухгалтер или ИИ (промпт для него: [pilot/scribe-prompt.md](pilot/scribe-prompt.md), шаблон: [pilot/draft-template.json](pilot/draft-template.json)). Черновик — обычный файл, **ключей в нём нет и полномочий он не даёт**.

Сначала посмотрите, что вам подсунули — ничего не подписывая:

```bash
bc apply --draft мой-профиль.json --dry-run
```

Команда печатает **каждую** запись целиком, со всеми тегами. Читайте. Это тот момент, ради которого всё устроено именно так.

### Шаг 3. Подписать своим ключом

```bash
bc apply --draft мой-профиль.json --via https://free-labour-agg.duckdns.org
```

Спросит `Подписать N запис(ь/и)? [y/N]`. Каждая запись становится отдельным блоком со **своей** подписью. Проверить, что получилось:

```bash
bc list
```

### Без писаря — можно и руками

Сначала найдите свой слаг в каталоге, чтобы не выдумывать:

```bash
bc catalog --via https://free-labour-agg.duckdns.org --search электрик
#   Ремонт и строительство
#     prof.electrician          Электрик
```

```bash
bc concept add "Электромонтаж, 8 лет, допуск до 1000В" \
    --tag kind:skill --tag cat:prof.electrician \
    --tag geo:55.75,37.62 --tag r:30 --tag grade:4 --tag exp:8 \
    --via https://free-labour-agg.duckdns.org
```

**Неизвестный слаг система не примет** — ни от вас, ни от ИИ-писаря. Опечатка вроде `prof.elektrik` не падает громко: вы бы просто перестали находиться по этому навыку. Поэтому она отвергается сразу, с подсказкой похожих. Если нужного слага в каталоге ещё нет — `--force`, и скажите организатору дополнить каталог.

## Часть 2а. Найти людей

Внешние инструменты не нужны — всё через `bc`:

```bash
bc directory --skill prof.electrician --via https://free-labour-agg.duckdns.org   # кто умеет
bc needs list --need need.electrical  --via https://free-labour-agg.duckdns.org   # кому нужно
bc needs list --via https://free-labour-agg.duckdns.org                           # все потребности
```

Закрытые потребности из выдачи выпадают сами.

## Часть 2б. Сделка: от «берусь» до расчёта трудом

Когда стыковка нашла вас друг другу, сделка ведётся командой `bc deal` — она сама подставляет все ссылки, ничего копировать не надо:

```bash
bc deal list --via URL            # мои сделки: стадия и ГОТОВАЯ следующая команда
```

Каждая стадия печатает «→ ваш ход» с уже заполненными ссылками. Полный путь:

```bash
# Исполнитель увидел потребность и берётся (навык подставится сам):
bc deal take НУЖДА_REF --via URL

# Заказчик нанимает (обещание трудочасов с именем исполнителя):
bc deal hire НУЖДА_REF --executor UID --units 6 --via URL

# Исполнитель сделал работу — записывается и привязывается к потребности разом:
bc deal work НУЖДА_REF --hours 6 --action "Замена проводки" --via URL

# Заказчик принимает (команды напечатает bc deal list) и рассчитывается:
bc deal settle НУЖДА_REF --via URL
```

`deal settle` платит по приёмке, гасит ваше обещание и **спрашивает**, закрыта ли потребность, — потому что оплата не всегда значит «закрыто» (бывает частичная оплата и второй этап). Решаете вы, не программа.

Каждый шаг — это подписанные блоки в вашей цепи; обещание, работа, приёмка и перевод видны потом в `bc deal list`, `bc pledge list`, `bc wallet`.

## Часть 3. Когда потребность закрыта

Нашли исполнителя, работа сделана — снимите потребность, чтобы она не висела в списке. Узнайте хеши своих записей:

```bash
bc list
```

Соберите черновик со связью «закрыто» (`ВАША_ЦЕПЬ` — из `bc identity show`):

```json
{"records":[
  {"t":"concept-link",
   "from":"ВАША_ЦЕПЬ/ХЕШ_ЛЮБОЙ_ВАШЕЙ_ЗАПИСИ",
   "to":"ВАША_ЦЕПЬ/ХЕШ_ЗАКРЫВАЕМОЙ_ПОТРЕБНОСТИ",
   "kind":"закрыто"}
]}
```

```bash
bc apply --draft close.json --via https://free-labour-agg.duckdns.org
```

Потребность пометится `"closed": true` и выпадет из стыковки. **Закрыть запись может только её хозяин** — чужая связь игнорируется.

## Безопасность: что здесь защищено и чем

- **Черновик не несёт полномочий.** В нём нет ключей. Его можно слать в мессенджере и генерировать коммерческим ИИ — до вашей подписи он просто текст.
- **Подписываете только вы.** Один блок — одна подпись вашим ключом. «Подписать пачку одной подписью» физически невозможно.
- **Перевод трудочасов в черновик не пролезет.** `bc apply` отвергает `transfer`, `pledge`, `acceptance` и прочие ценностные записи — **весь черновик целиком**, даже если остальное безобидно. Так перевод не спрячется среди фактов профиля.
- **Всегда делайте `--dry-run` перед подписью.** Это две секунды.
- **Геолокация — ваша ручка приватности.** Точность координаты выбираете вы: `geo:55.7` — это район, `geo:55.7512` — это дом. Хотите обезличиться — огрубите координату и увеличьте радиус `r:`. Консультации и удалённая работа — `remote:yes` и `r:global`, координата тогда не нужна вовсе.

## Промпт для стыковки (организатору)

Отдайте ИИ файл `profiles.json` и попросите:

> Вот выгрузка профилей участников из блокчейна. Каждый участник (`chain`) имеет `skills` (что умеет), `needs` (что нужно), `aspirations` (чему готов научиться). У фактов есть теги: `cat:` — запись каталога, `geo:` — координата, `r:` — радиус в км (или `global`), `remote:yes` — можно удалённо, `grade:` — разряд 1–6, `urgency:` — срочность.
>
> Соответствие «потребность → закрывающая профессия» возьми из `docs/catalogs/needs.json`, поле `closed_by`.
>
> Задачи:
> 1. Для каждой потребности найди участников, чей навык её закрывает и кто **досягаем**: расстояние между координатами не больше радиуса обоих. `remote:yes` или `r:global` — досягаем всегда.
> 2. Отдельно выпиши потребности, которые **никто не закрывает** — это дефицит.
> 3. Для каждого дефицита предложи, кто мог бы переучиться: смотри `aspirations` и тег `retrain:yes`.
> 4. Найди **кольца обмена** (A закрывает B, B закрывает C, C закрывает A) — они самые ценные: замыкаются без внешнего долга.
>
> Не выдумывай участников и не додумывай навыки, которых нет в данных.

**Что это дало на живом прогоне впятером:** 5 из 6 потребностей закрылись, нашлось кольцо из трёх человек (швея → электрик → повар → швея) и одна пара, а единственный дефицит (ремонт бытовой техники) совпал с человеком, который как раз заявил готовность переучиться.
