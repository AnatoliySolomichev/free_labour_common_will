# Симуляция экономики сообщества (`sim/`)

Расширенный интеграционный тест-симуляция: пять человек разных специальностей
проживают «месяц» (30 дней), производят продукцию, обменивают её на эквивалент
затраченных трудо-часов, эмитируют и погашают именной взаимный кредит и
используют остальные функции проекта. Гоняются **настоящие бинарники**
(`bc` + `aggregator_server`) — это не юнит-тест на моках, а прогон системы как её
видит пользователь.

## Запуск

```bash
cmake --build build            # нужны build/modules/cli/bc и .../aggregator_server
apt-get install -y libfaketime # для управления «датой» у клиентов
NDAYS=30 sim/economy_month.sh
```

Параметры через env: `NDAYS` (дней), `PORT`, `START` (дата дня 0), `WORK`
(рабочая директория), `ALPHA` (сглаживание ставок). Результаты — в
`$WORK/results/` (по умолчанию `/tmp/economy_sim/results`). Пример готового
прогона лежит в [`sample_results/`](sample_results/).

## Действующие лица и торговый граф

| Кто   | Специальность | Разряд | Производит | Покупает у |
|-------|---------------|:------:|------------|------------|
| Анна  | Фермер        | 3      | зерно, молоко | Глеб (ремонт), Борис (хлеб) |
| Борис | Пекарь        | 3      | хлеб (нужно зерно) | Анна (зерно), все |
| Вера  | Плотник       | 4      | мебель, ремонт | Анна, Глеб |
| Глеб  | Механик       | 2 → 3  | ремонт техники | Вера (мебель), Борис (хлеб) |
| Дарья | Врач          | 5      | лечение (услуга) | Борис, Вера |

Встречные сделки закольцованы, поэтому именная бумага **циркулирует и гасится**
(долг гасится встречным трудом), а не только накапливается.

## Что задействовано (функции проекта)

- **Идентичность / ветки** — `identity create`, ветка по умолчанию.
- **Труд** — `specialty add`, `grade add` (в т.ч. повышение разряда 2→3 в день 15),
  `work log` с входами/выходами и серийниками.
- **Обмен** — `fetch` → `accept` (оценка = ставка × k × часы) → `pay`
  (самоэмиссия / трата чужой бумаги) → `transfer recv` (Copy-ack + Redemption).
- **Ставки труда** — реальный `build_daily_rates` через `/economy/rates?day=…`;
  скрипт сглаживает их (перенос вчерашней + EMA) и подаёт как базу оценки на
  следующий день — замыкая петлю §2а.
- **Свидетельствование** — `seal add` (коллега рядом пломбирует work-запись).
- **Слияния** — `merge run` / `merge serve` через релей → граф discovery.
- **Доска идей** — `concept add`, `pledge add`/`revoke`, `react`, `copy`,
  расчёт по пледжам через `transfer send --reason <pledge>`; демонстрируются все
  четыре статуса пледжа (погашен / погашен / отозван / истёк).
- **Экономические досье** — `/economy/chain/:uid`, `/economy/ideas`, `/discovery/:uid`.
- **Ротация ключа** — `revoke create --replace` + `revoke verify` + `revoke status` (§6.7).

## Итоги примера прогона (30 дней, `sample_results/`)

- **88 заверенных сделок**, 265 трудо-часов, 294 блока в сети.
- **Инвариант взаимного кредита сходится точно**: Σ(эмитировано − погашено) =
  70.89 ч = Σ бумаги на руках у держателей. Долги = требования, аналитически.
- **Ставки сложились рынком**: Врач +19 %, Плотник +20 % (дефицит/качество
  дорожают), Пекарь −28 % (ежедневный конкурентный стейпл дешевеет), Фермер ≈ 0 %
  (сезонность). Механик рос через разряд, а не через k.
- **Доска идей**: «общая мельница» — 35 ч труда погашено, 4 спонсора, 70 реакций.
- **Граф доверия**: 4 слияния сплели цепь-путь Анна–Борис–Вера–Глеб–Дарья.
- **Ротация ключа**: самопроверяемый сертификат, ветка `REPLACED`, 63 блока чисто.

## Замечание (найдено симуляцией)

`revoke create --replace` основного кошелька ломает **локальный** `compute_wallet`:
`Context::local_branches()` начинает отдавать ветку дважды (файл ключа-замены
парсится как отдельный индекс), поэтому `issued`/holdings в `bc wallet`
удваиваются. Авторитетные источники (нить эмиссии цепи и `/economy/chain`
агрегатора) при этом совпадают и верны. В симуляции статистика снимается **до**
эпилога-ротации, чтобы цифры кошельков были чистыми. Кандидат на баг-репорт:
дедуплицировать ветки в `local_branches()` или исключать ветки-замены.
